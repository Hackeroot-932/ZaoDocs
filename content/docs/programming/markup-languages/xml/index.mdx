## 1. Introduction to Markup Languages

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <h3 className="text-xl font-semibold mb-4 text-gray-800 dark:text-white">Digital Information Representation</h3>

  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
    <div className="p-4 bg-blue-50 dark:bg-blue-900/30 rounded-lg">
      <h4 className="font-semibold text-blue-700 dark:text-blue-300 mb-2">Computer Data Storage</h4>
      <ul className="list-disc pl-5">
        <li>Computers store all information in binary format (0s and 1s)</li>
        <li>Different types of data (text, images, sound, video) require specific encoding methods</li>
        <li>Text is encoded using character encoding standards (e.g., ASCII, Unicode/UTF-8)</li>
      </ul>
    </div>
    
    <div className="p-4 bg-green-50 dark:bg-green-900/30 rounded-lg">
      <h4 className="font-semibold text-green-700 dark:text-green-300 mb-2">Evolution of Markup Languages</h4>
      <ul className="list-disc pl-5">
        <li><strong>Charles Goldfarb</strong>: Considered the father of markup languages</li>
        <li><strong>GML</strong>: Initial IBM markup language that laid the foundation for SGML</li>
        <li><strong>SGML</strong>: Standard Generalized Markup Language, the parent of HTML and XML</li>
        <li><strong>HTML</strong>: Created by Tim Berners-Lee for web document structure</li>
        <li><strong>XML</strong>: Developed as a simplified version of SGML for data storage and transport</li>
      </ul>
    </div>
  </div>
</div>

## 2. What is XML?

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <p className="mb-4">XML (Extensible Markup Language) is a markup language and file format for storing, transmitting, and reconstructing arbitrary data. It defines a set of rules for encoding documents in a format that is both human-readable and machine-readable.</p>

  <div className="grid grid-cols-1 md:grid-cols-2 gap-4 mb-4">
    <div className="p-4 bg-blue-50 dark:bg-blue-900/30 rounded-lg">
      <h4 className="font-semibold text-blue-700 dark:text-blue-300 mb-2">Key Features</h4>
      <ul className="list-disc pl-5 text-gray-600 dark:text-gray-300">
        <li>Platform and language independent</li>
        <li>Self-descriptive syntax</li>
        <li>Strict rules for parsing</li>
        <li>Hierarchical data structure</li>
        <li>Unicode support for all languages</li>
      </ul>
    </div>
    <div className="p-4 bg-green-50 dark:bg-green-900/30 rounded-lg">
      <h4 className="font-semibold text-green-700 dark:text-green-300 mb-2">Common Uses</h4>
      <ul className="list-disc pl-5 text-gray-600 dark:text-gray-300">
        <li>Data exchange between systems</li>
        <li>Configuration files</li>
        <li>Web services (SOAP)</li>
        <li>Document storage</li>
        <li>Database data transfers</li>
      </ul>
    </div>
  </div>

  ```xml
  <?xml version="1.0" encoding="UTF-8"?>
  <bookstore>
    <book category="fiction">
      <title>The Great Gatsby</title>
      <author>F. Scott Fitzgerald</author>
      <year>1925</year>
      <price>9.99</price>
    </book>
  </bookstore>
  ```
</div>

## 4. XML Document Structure

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">
    <div className="space-y-4">
      <h4 className="font-semibold mb-2">Basic Structure</h4>
      <ul className="list-disc pl-5">
        <li><strong>Prolog</strong>: XML declaration and processing instructions</li>
        <li><strong>Root Element</strong>: Single top-level element containing all others</li>
        <li><strong>Child Elements</strong>: Nested elements forming the document hierarchy</li>
        <li><strong>Attributes</strong>: Additional element properties</li>
        <li><strong>Text Content</strong>: Actual data within elements</li>
      </ul>
      
      ```xml
      <?xml version="1.0" encoding="UTF-8"?>
      <root>
        <child attribute="value">
          <grandchild>Content</grandchild>
        </child>
      </root>
      ```
    </div>
    
    <div className="space-y-4">
      <h4 className="font-semibold mb-2">Namespaces</h4>
      <p>XML namespaces provide a way to avoid element name conflicts by using unique identifiers:</p>
      
      ```xml
      <root xmlns:h="http://www.w3.org/TR/html4/"
            xmlns:f="http://www.example.com/furniture">
        <h:table>HTML table</h:table>
        <f:table>Furniture table</f:table>
      </root>
      ```
    </div>
  </div>
</div>

## 5. Well-Formed XML

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <h3 className="text-xl font-semibold mb-4 text-gray-800 dark:text-white">Rules for Well-Formed XML</h3>

  <div className="grid grid-cols-1 md:grid-cols-2 gap-4">
    <div className="p-4 bg-blue-50 dark:bg-blue-900/30 rounded-lg">
      <h4 className="font-semibold text-blue-700 dark:text-blue-300 mb-2">Element Rules</h4>
      <ul className="list-disc pl-5 text-gray-600 dark:text-gray-300">
        <li>Must have a single root element</li>
        <li>All elements must be properly nested</li>
        <li>Tags are case-sensitive</li>
        <li>All elements must be closed</li>
        <li>Empty elements can use self-closing tags</li>
      </ul>
    </div>
    <div className="p-4 bg-green-50 dark:bg-green-900/30 rounded-lg">
      <h4 className="font-semibold text-green-700 dark:text-green-300 mb-2">Attribute Rules</h4>
      <ul className="list-disc pl-5 text-gray-600 dark:text-gray-300">
        <li>Must be quoted (single or double quotes)</li>
        <li>Cannot contain multiple values</li>
        <li>Cannot contain tree structures</li>
        <li>Cannot contain markup characters</li>
        <li>Must have unique names within an element</li>
      </ul>
    </div>
  </div>
</div>

## 6. XML Validation

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <Tabs items={["DTD", "XML Schema", "Tools"]}>
    <Tab>
      <div className="p-4 border rounded-lg bg-white dark:bg-gray-800 my-4">
        <h4 className="font-bold mb-2 text-blue-600 dark:text-blue-400">Document Type Definition (DTD)</h4>
        <p className="mb-4">DTDs define the structure and legal elements of an XML document:</p>
        
        ```dtd
        <!ELEMENT bookstore (book+)>
        <!ELEMENT book (title, author, year, price)>
        <!ATTLIST book category CDATA #REQUIRED>
        <!ELEMENT title (#PCDATA)>
        <!ELEMENT author (#PCDATA)>
        <!ELEMENT year (#PCDATA)>
        <!ELEMENT price (#PCDATA)>
        ```
      </div>
    </Tab>
    <Tab>
      <div className="p-4 border rounded-lg bg-white dark:bg-gray-800 my-4">
        <h4 className="font-bold mb-2 text-green-600 dark:text-green-400">XML Schema (XSD)</h4>
        <p className="mb-4">XML Schema provides a more powerful way to define XML structure:</p>
        
        ```xml
        <xs:schema>
          <xs:element name="bookstore">
            <xs:complexType>
              <xs:sequence>
                <xs:element name="book" maxOccurs="unbounded">
                  <xs:complexType>
                    <xs:sequence>
                      <xs:element name="title" type="xs:string"/>
                      <xs:element name="author" type="xs:string"/>
                      <xs:element name="year" type="xs:integer"/>
                      <xs:element name="price" type="xs:decimal"/>
                    </xs:sequence>
                    <xs:attribute name="category" type="xs:string"/>
                  </xs:complexType>
                </xs:element>
              </xs:sequence>
            </xs:complexType>
          </xs:element>
        </xs:schema>
        ```
      </div>
    </Tab>
