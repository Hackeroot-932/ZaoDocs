## 2. String Methods: Length and Search

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <h3 className="text-xl font-semibold mb-4 text-gray-800 dark:text-white">Finding String Length and Content</h3>

  ```js
  const producto = "Monitor de 20 pulgadas";

  // Get string length
  console.log(producto.length); // 22

  // Check if string includes a substring
  console.log(producto.includes("Monitor")); // true
  console.log(producto.includes("Tablet")); // false
  console.log(producto.includes("monitor")); // false (case sensitive)

  // Find position of substring
  console.log(producto.indexOf("Monitor")); // 0
  console.log(producto.indexOf("Tablet")); // -1 (not found)
  ```

  <div className="mt-4 space-y-2">
    <p className="text-gray-600 dark:text-gray-300">
      The <code>length</code> property returns the total number of characters in a string, including spaces and special characters.
    </p>
    <p className="text-gray-600 dark:text-gray-300">
      The <code>includes()</code> method performs a case-sensitive search to determine whether a string contains another string.
    </p>
    <p className="text-gray-600 dark:text-gray-300">
      The <code>indexOf()</code> method returns the position of the first occurrence of a substring, or -1 if not found.
    </p>
  </div>
</div>

## 4. String Transformation Methods

<div className="p-6 bg-white dark:bg-gray-800 rounded-xl shadow-sm mb-8">
  <h3 className="text-xl font-semibold mb-4 text-gray-800 dark:text-white">Modifying String Content</h3>

  ```js
  const producto = "           Monitor de 20 pulgadas           ";

  // 1. Trimming whitespace
  console.log(producto.trimStart()); // Remove leading spaces
  console.log(producto.trimEnd());   // Remove trailing spaces
  console.log(producto.trim());      // Remove both leading and trailing spaces

  // 2. Replacing content
  const texto = "Monitor 20 pulgadas";
  console.log(texto.replace("pulgadas", '"')); // 'Monitor 20 "'
  console.log(texto.replace("Monitor", 'Monitor Curvo')); // 'Monitor Curvo 20 pulgadas'

  // 3. Slicing strings
  console.log(texto.slice(0, 10));  // "Monitor 20"
  console.log(texto.slice(8));      // "20 pulgadas"
  console.log(texto.substring(0, 10)); // "Monitor 20"
  
  // 4. Repeating strings
  const texto2 = " en rebaja".repeat(3);
  console.log(`${texto} ${texto2}`); // "Monitor 20 pulgadas en rebaja en rebaja en rebaja"
  ```

  <div className="mt-6 p-6 bg-gradient-to-r from-purple-50 to-pink-50 dark:from-purple-900/30 dark:to-pink-900/30 rounded-xl">
    <h4 className="text-lg font-semibold mb-3 text-purple-800 dark:text-purple-200">Understanding slice() vs substring()</h4>
    
    ```js
    const producto = "Monitor 20 pulgadas";

    // Basic usage - both methods work similarly
    console.log(producto.slice(0, 7));     // "Monitor"
    console.log(producto.substring(0, 7));  // "Monitor"

    // Key Difference 1: Handling negative indices
    console.log(producto.slice(-9));        // "pulgadas" (counts from end)
    console.log(producto.substring(-9));    // "Monitor 20 pulgadas" (treats negative as 0)

    // Key Difference 2: Handling start > end
    console.log(producto.slice(2, 1));      // "" (returns empty string)
    console.log(producto.substring(2, 1));  // "o" (swaps arguments)

    // Key Difference 3: End index behavior
    console.log(producto.slice(8));         // "20 pulgadas" (to the end)
    console.log(producto.substring(8));     // "20 pulgadas" (to the end)
    ```

    <div className="mt-4 grid grid-cols-1 md:grid-cols-2 gap-4">
      <div className="p-4 bg-white/50 dark:bg-gray-800/50 rounded-lg">
        <h5 className="font-semibold text-purple-700 dark:text-purple-300 mb-2">slice() Method</h5>
        <ul className="list-disc list-inside text-gray-600 dark:text-gray-300 space-y-1">
          <li>Supports negative indices</li>
          <li>Returns empty string if start > end</li>
          <li>More similar to array slicing</li>
          <li>Preferred for modern JavaScript</li>
        </ul>
      </div>
      <div className="p-4 bg-white/50 dark:bg-gray-800/50 rounded-lg">
        <h5 className="font-semibold text-pink-700 dark:text-pink-300 mb-2">substring() Method</h5>
        <ul className="list-disc list-inside text-gray-600 dark:text-gray-300 space-y-1">
          <li>Treats negative indices as 0</li>
          <li>Swaps arguments if start > end</li>
          <li>Legacy method from early JavaScript</li>
          <li>Less predictable behavior</li>
        </ul>
      </div>
    </div>

    <div className="mt-4 p-4 bg-yellow-50/50 dark:bg-yellow-900/30 rounded-lg">
      <h5 className="font-semibold text-yellow-800 dark:text-yellow-200 mb-2">ü§î Why the Difference?</h5>
      <p className="text-gray-600 dark:text-gray-300">
        When <code>substring(2,1)</code> returns "o", it's because substring automatically swaps the arguments if start > end, effectively becoming <code>substring(1,2)</code>. In contrast, <code>slice(2,1)</code> follows a more logical approach: if the starting point is after the ending point, it returns an empty string.
      </p>
    </div>
  </div>

  <div className="mt-6 grid grid-cols-1 md:grid-cols-2 gap-4">
    <div className="p-4 bg-blue-50 dark:bg-blue-900/30 rounded-lg">
      <h4 className="font-semibold text-blue-700 dark:text-blue-300 mb-2">Trim Methods</h4>
      <p className="text-gray-600 dark:text-gray-300">
        The trim methods are essential for cleaning user input and ensuring consistent string formatting. They remove unwanted whitespace without modifying the actual content.
      </p>
    </div>
    <div className="p-4 bg-green-50 dark:bg-green-900/30 rounded-lg">
      <h4 className="font-semibold text-green-700 dark:text-green-300 mb-2">Slice vs Substring</h4>
      <p className="text-gray-600 dark:text-gray-300">
        While both <code>slice()</code> and <code>substring()</code> extract parts of a string, <code>slice()</code> supports negative indices and is generally more flexible.
      </p>
    </div>
  </div>
</div>

  // Practical example: URL parsing
